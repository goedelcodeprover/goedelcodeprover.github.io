Introduction

Problem. Large language models (LLMs) can generate plausible code, yet they provide no guarantee of formal correctness—subtle logical errors and specification violations remain prevalent. Automated code verification addresses this gap by constructing machine-checkable proofs that implementations satisfy formal specifications. We study this problem in Lean 4, a modern interactive theorem prover.

Method. We propose a hierarchical proof search framework with two stages: lemma decomposition, which breaks a complex verification goal into simpler subgoals, and lemma completion, which discharges each subgoal through iterative proof generation. A principled decomposition score guides both training and inference, ensuring alignment between optimization and deployment.

Model Training & Release. We train a single unified 8B model for both decomposition and completion via supervised initialization followed by hybrid reinforcement learning, using the decomposition score as a dense reward signal. We publicly release our trained model along with an evaluation framework to facilitate future research in automated code verification.

Results. Experiments on multiple Lean-based code verification benchmarks, including Verina, Clever, and AlgoVeri, demonstrate the effectiveness of our approach.

Technical Overview

Key Insight: Lemma Decomposition + Lemma Completion

Effective Lemma Decomposition. We score decompositions along two axes. Constructive justification: the decomposed lemma must be provable, and we run quickcheck on candidate lemmas; any counterexample discards the decomposition. Decomposition effectiveness: we measure it via an operator-footprint metric. The score is used both as the RL reward and as the ranking criterion at inference, enabling efficient search and pruning.

Iterative Lemma Completion. For each decomposed lemma, the LLM generates a candidate proof and submits it to Lean. The compiler either accepts the proof or returns an error. When it fails, the model uses the compiler feedback (e.g., type mismatch, unresolved goal) to revise the proof and tries again. This iterative loop mimics how humans fix failed proofs and is also integrated into both supervised fine-tuning and reinforcement learning.

Training Pipeline

We initialize a supervised model on scaffolded decomposition and completion data. Then we run hybrid RL: GRPO optimizes decomposition using the decomposition score as reward, while we replay high-quality completion trajectories (SFT) to stabilize proof generation. Both losses update the same policy. The problem set is augmented with online lemmas from rollouts, forming a self-generated curriculum.

Inference Pipeline

We use a single model for both stages. (1) Decomposition: for a given theorem, the LLM proposes sub-lemmas with a constructive proof that Lean verifies; we keep the best-scoring decomposition and recurse until remaining goals are simple. (2) Completion: each leaf lemma is proved by the same policy in an iterative loop: generate a candidate proof, then refine it using Lean compiler feedback.
