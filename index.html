Introduction

Problem. Large language models (LLMs) can generate syntactically plausible code, yet they provide no guarantee of formal correctness; subtle logical errors and specification violations remain prevalent. Automated code verification seeks to address this gap by constructing machine-checkable proofs that implementations satisfy formal specifications in Lean 4.

Method. We propose a hierarchical two-stage proof search framework: first decomposing a complex verification goal into structurally simpler subgoals, and then discharging each subgoal through iterative proof generation and refinement. To guide decomposition, we introduce a principled scoring mechanism that enforces constructive justification with QuickCheck-style filtering and promotes structural simplification via an operator-footprint metric. The same score is used consistently for both training and inference, ensuring alignment between optimization and deployment. A single unified policy is trained through supervised initialization followed by hybrid reinforcement learning.

Results. Experiments on multiple Lean-based code verification benchmarks, including Verina, Clever, and AlgoVeri, demonstrate the effectiveness and robustness of our approach.

Our Methodology

Two-Stage Framework

We use a single model for both stages. (1) Decomposition: for a given theorem, the LLM proposes sub-lemmas with a constructive proof that Lean verifies; we keep the best-scoring decomposition and recurse until remaining goals are simple. (2) Completion: each leaf lemma is proved by the same policy in an iterative loop: generate a candidate proof, then refine it using Lean compiler feedback.

Effective Lemma Decomposition

We score decompositions along two axes. Constructive justification: the decomposed lemma must be provable, and we run quickcheck on candidate lemmas; any counterexample discards the decomposition. Decomposition effectiveness: we measure it via an operator-footprint metric. The score is used both as the RL reward and as the ranking criterion at inference, enabling efficient search and pruning.

Iterative Lemma Completion

For each decomposed lemma, the LLM generates a candidate proof and submits it to Lean. The compiler either accepts the proof or returns an error. When it fails, the model uses the compiler feedback (e.g., type mismatch, unresolved goal) to revise the proof and tries again. This iterative loop mimics how humans fix failed proofs and is also integrated into both supervised fine-tuning and reinforcement learning.

Training: SFT + Hybrid RL

We initialize a supervised model on scaffolded decomposition and completion data. Then we run hybrid RL: GRPO optimizes decomposition using the decomposition score as reward, while we replay high-quality completion trajectories (SFT) to stabilize proof generation. Both losses update the same policy. The problem set is augmented with online lemmas from rollouts, forming a self-generated curriculum.
